Speech Recognition

• The SpeechRecognition interface of the Web Speech API is the controller interface for the recognition service; this also handles the SpeechRegonitionEvent sent from the recognition service.

• NOTE : On some browsers, like Chrome, using Speech Recognition on a web page involves a server-based recognition engine. Your audio is sent to a web service for recognition processing, so it won't work offline.

• Speech Recognition also inherits properties from its parent interface, EventTarget.

Speech Recognition()

• The SpeechRecognition() constructor creates a new SpeechRecognition object instance.

• SYNTAX : new SpeechRecognition()

• It does not have any parameters.

interimResults property

• The interimResults property of the SpeechRecognition interface controls whether interim results should be returned (true) or not (false). Interim results are results that are not yet final (e.g. the SpeechRecognitionResult. isFinal property is false). The default value for interimResults is false.

• A boolean value representing the state of the current SpeechRecognition's interim results. true means interim results are returned and false means they aren't.

SpeechRecognition: result event

• The result event of the Web Speech API is fired when the speech recognition service returns a result - a word or phrase has been positively recognized and this has been communicated back to the app.

• SYNTAX : Use the event name in methods like addEventListener(), or set an event handler property.
addEventListener('result', event => {})
onresult = event => {}

• We can also use the 'onresult' event handler property.

SpeechRecognition: end event

• The end event of the Web Speech API SpeechRecognition object is fired when the speech recognition service has disconnected.

• SYNTAX : Use the event name in methods like addEventListener, or set an event handler property.
addEventListener('end', event => {})
onend = event => {}

• We can also use the 'onend' event in addEventListener method.
